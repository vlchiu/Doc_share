################################ Tạo container mới ######################################################################

# Trên server có sẵn image hsptek:v1 rồi nên không cần build images nữa
- Tạo 1 container với images có sẵn hsptek:v1, mount vào ổ chức source code hlos vừa lấy về
	docker run -it --privileged -v /media/bsp/data/hlos:/home/hsptek/source hsptek:v1
		Lưu ý: /media/bsp/data/hlos là đường dẫn source code ubuntu trên máy tính
				/home/hsptek/source là đường dẫn source ở trên docker
- Sau khi chạy docker run, terminal sẽ truy cập vào shell trên docker,
	cd source -> Truy cập vào source code hlos

Lưu ý chỉ chạy docker run 1 lần duy nhất (mỗi lần chạy sẽ tạo ra 1 container id khác, không cần thiết) để tạo container, sau đó để truy cập vào container đó thì ta dùng lệnh :

	docker ps -a    # tìm container đang up, check container ID
	docker exec -it <container_id> bash

- Tiến hành setup môi trường và build như mục "4. Setup docker build enviroment (lưu ý các lệnh dưới đều chạy ở trong môi trường docker)"
- Chạy build theo hướng dẫn mục "6. Build"

- Để build lại 1 gói package nào đó ta dùng lệnh:
	bitbake -v -fc clean <package> # clean đi dữ liệu cũ
	bitbake -v <package> # build lại

Ví dụ: sau khi chỉnh sửa 1 số package, cần build lại full OS để cập nhật vào bản images cuối, ta dùng lệnh:
	bitbake -v -fc clean qti-ubuntu-robotics-image; bitbake qti-ubuntu-robotics-image
